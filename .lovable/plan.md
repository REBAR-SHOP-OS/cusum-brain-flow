

# Summarize → PDF Report Generation

## What
When the user clicks "Summarize" in the AI Processing toolbar, instead of (or in addition to) showing the summary inline, generate a detailed PDF report of the conversation and offer it as a downloadable file. The PDF must include an AI disclaimer: "This report was generated by AI and may contain errors."

## Changes

### 1. `src/components/transcribe/PostProcessToolbar.tsx`
- Modify the "Summarize" button behavior: after calling `transcribe-translate` with `postProcess: "summarize"`, take the result and generate a PDF client-side using `jspdf`
- PDF content structure:
  - Header: "Conversation Report" + date/time
  - Participants: `{selectedSpeaker} ↔ NEEL` (if a speaker is selected) or "General Transcript"
  - Full transcript section (original text)
  - AI Summary section (the summarized result)
  - Footer disclaimer: "⚠️ This report was generated by Artificial Intelligence and may contain inaccuracies."
- Auto-download the PDF after generation
- Still show the summary inline via `onResult`

### 2. Install `jspdf`
- Add `jspdf` package for client-side PDF generation

### 3. Props update
- Pass `selectedSpeaker` from `TranscribeView` to `PostProcessToolbar` so the PDF can include who the conversation is between
- Pass committed transcripts for the full conversation log

### Technical Notes
- Using client-side PDF generation (`jspdf`) — no edge function needed
- PDF includes: title, date, participants, transcript, AI summary, disclaimer
- RTL text (Farsi/Arabic) may not render perfectly in jspdf but English summary will be clean
