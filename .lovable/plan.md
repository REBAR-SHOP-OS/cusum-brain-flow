

# AI Video Director — Alibaba Compute Engine + Brand Kit Save

## 1. Brand Kit Save Button

The `brand_kit` table exists but the Ad Director's `BrandProfile` type has different fields (name, website, tagline, cta, targetAudience, primaryColor, secondaryColor, etc.) vs the table's schema (business_name, brand_voice, description, value_prop, colors JSONB). Need to bridge these.

### Changes
- **`ScriptInput.tsx`**: Add a "Save Brand Kit" button below the brand preview strip that persists the current `BrandProfile` to the `brand_kit` table
- **`AdDirectorContent.tsx`**: On mount, load saved brand kit from DB and map it to `BrandProfile` fields. Add the `useBrandKit` hook integration.
- **Field mapping**: `brand.name` → `business_name`, `brand.tagline + cta` → `value_prop`, `brand.targetAudience` → `description`, `brand.primaryColor/secondaryColor` → `colors.primary/secondary`, `brand.logoUrl` → `logo_url`, `brand.website + fontStyle + referenceAesthetic` → `brand_voice` (JSON-encoded extras)

### Migration
Add columns to `brand_kit` for the Ad Director fields that don't fit current schema:
```sql
ALTER TABLE public.brand_kit 
  ADD COLUMN IF NOT EXISTS website TEXT DEFAULT '',
  ADD COLUMN IF NOT EXISTS tagline TEXT DEFAULT '',
  ADD COLUMN IF NOT EXISTS cta TEXT DEFAULT '',
  ADD COLUMN IF NOT EXISTS target_audience TEXT DEFAULT '',
  ADD COLUMN IF NOT EXISTS font_style TEXT DEFAULT '',
  ADD COLUMN IF NOT EXISTS reference_aesthetic TEXT DEFAULT '';
```

This lets us do a clean 1:1 save/load without JSON-packing fields.

## 2. Alibaba as Primary Compute Engine

The video **generation** already defaults to Wan 2.6 (`provider: "wan"`, `model: "wan2.6-t2v"`). The user wants Alibaba for the compute/assembly side too.

DashScope does **not** have a video merging/concatenation API. However, we can improve the pipeline by:

### Approach: Server-Side Assembly via Edge Function
Create a new edge function `assemble-video` that:
1. Takes an array of scene video URLs (already hosted/generated by Wan)
2. Uses DashScope's video-to-video capabilities to create transitions between scenes
3. For final assembly, since DashScope can't concat, use a hybrid approach:
   - Generate each scene via Wan (already done)
   - Use the **Wan keyframe-to-video** API (`wan2.1-kf2v-plus`) to create smooth transitions between the last frame of scene N and first frame of scene N+1
   - Store all clips in storage, then do client-side stitch with the improved `videoStitch.ts`

Actually, the more practical approach: since DashScope lacks concat, we should label the engine properly and make the architecture explicit. The compute engine selection should show "Alibaba Wan 2.6" as the render engine everywhere, and the assembly step should be clearly labeled as "Browser Assembly" with a note that server-side MP4 export requires an external service.

### What I'll actually implement:
- **Default provider enforcement**: Ensure Ad Director always uses `provider: "wan"` for all scene generation (already done, but make it explicit in the model settings)
- **Label the engine**: Show "Compute Engine: Alibaba Wan 2.6" badge in the progress bar and FinalPreview
- **Scene Intelligence**: Always show "Alibaba Wan 2.6" as the render engine in scene cards
- **AdvancedModelSettings**: Add a "Compute Engine" selector showing Wan 2.6 as default with Veo/Sora as fallbacks
