

## شناسایی محل سه ابعاد اضافی

طبق تصویر، در bottom selector روی preview ویدئو شش نسبت نمایش داده می‌شود:
`16:9` | `9:16` | `1:1` | **`4:3`** | **`4:5`** | **`21:9`** ← سه مورد دور قرمز

## محل دقیق در کد

**فایل:** `src/components/ad-director/ProVideoEditor.tsx` خط 213-220

```ts
const ASPECT_RATIOS: Record<string, string> = {
  "16:9": "16/9",
  "9:16": "9/16",
  "1:1": "1/1",
  "4:3": "4/3",      ← حذف
  "4:5": "4/5",      ← حذف
  "21:9": "21/9",    ← حذف
};
```

این object دو جا استفاده می‌شود:
1. **خط 1887** — تعیین aspect-ratio container preview
2. **خط 2084** — render دکمه‌های selector که در تصویر دیده می‌شوند

با حذف سه کلید، **هر دو جا** خودکار به‌روز می‌شوند (popover فقط سه دکمه نمایش می‌دهد).

## تغییر دوم — Render-size map

خط 224-229 یک map دیگر هم وجود دارد (`RENDER_SIZES` یا مشابه) که سه ابعاد حذف‌شده را شامل می‌شود:
```ts
"4:3": [1080, 810],
"4:5": [1080, 1350],
"21:9": [1260, 540],
```
این‌ها هم برای تمیزی حذف می‌شوند (دیگر هیچ‌جا فراخوانی نمی‌شوند چون selector آن‌ها را offer نمی‌کند).

## تغییر سوم (اختیاری ولی هم‌راستا) — `VideoParameters.tsx`

خط 24:
```ts
const RATIOS = ["21:9", "16:9", "4:3", "1:1", "3:4", "9:16", "Smart"];
```
این کامپوننت در صفحه‌ی **پیش از editor** (تنظیمات تولید ویدئو اولیه) استفاده می‌شود. کاربر گفت «روی ویدئو همیشه فقط سه ابعاد» → این لیست هم باید همگام شود:
```ts
const RATIOS = ["16:9", "9:16", "1:1"];
```
(حذف `21:9`, `4:3`, `3:4`, `Smart`)

## آنچه دست‌نخورده می‌ماند
- منطق export، stitching، canvas drawing — همگی از مقدار state `aspectRatio` استفاده می‌کنند که حالا فقط می‌تواند یکی از سه مقدار معتبر باشد
- default = `"16:9"` (خط 205) همچنان معتبر
- DB / RLS / edge functions — هیچ تغییری
