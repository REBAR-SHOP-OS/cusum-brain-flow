
Goal: make SMS feel conversational for the CEO while stopping the “echo” behavior where your own inbound texts are forwarded back to you as alerts.

What I found
- The repeated gray “New SMS from +14165870788 …” messages are not AI replies. They are CEO alert texts generated by `sendCeoSmsAlert(...)` inside `supabase/functions/ringcentral-sync/index.ts`.
- `ringcentral-sync` currently alerts on every inbound SMS, including messages sent from your own phone number, and it does this after upsert without checking whether the message was newly inserted. That can replay old/self messages.
- `vizzy-sms-reply/index.ts` explicitly skips your number (`+14165870788`), so Vizzy never answers you conversationally by SMS. That is why you see alerts but not a real back-and-forth reply.
- Spam filtering is working, but it only blocks spam from auto-reply/alerts. It does not solve the self-alert loop.

Implementation plan

1. Stop sending CEO alerts for the CEO’s own texts
- Update `supabase/functions/ringcentral-sync/index.ts`
- Before calling `sendCeoSmsAlert`, normalize `fromAddress` and skip if it matches the CEO number.
- Also skip if the message is outbound or otherwise internal/company-originated.

2. Only alert on newly seen inbound SMS
- In `ringcentral-sync/index.ts`, use the upsert result more carefully so alerts only fire for genuinely new inbound messages, not for every sync pass over existing rows.
- This prevents repeated replay alerts from older messages.

3. Let Vizzy reply conversationally to the CEO
- Update `supabase/functions/vizzy-sms-reply/index.ts`
- Replace the hard “skip CEO” rule with a special CEO mode:
  - allow replies to your number
  - do not treat your messages as spam
  - keep rate limiting/dedupe, but tune it so normal conversation still works
- Keep skipping other internal/company numbers to avoid system loops.

4. Make CEO replies sound like a real chat
- Strengthen the SMS prompt in `vizzy-sms-reply/index.ts` so CEO texts get:
  - direct answers first
  - short conversational tone
  - no “sales lead” framing
  - no robotic fallback language unless truly needed
- Add guidance like:
  - if CEO says “Hi”, respond naturally
  - if CEO asks an ops question, answer directly from available context
  - if unknown, say what’s missing instead of generic sales copy

5. Protect against double-triggering between webhook and cron
- Keep webhook as the fast path for live reply.
- In `ringcentral-sync/index.ts`, make cron a catch-up path only for missed inbound messages, not a second notification source for already-processed texts.
- Use existing metadata/source fields where possible instead of adding schema changes.

Files to update
- `supabase/functions/ringcentral-sync/index.ts`
- `supabase/functions/vizzy-sms-reply/index.ts`

Expected result after fix
- Your own texts to Vizzy will no longer be bounced back to you as “New SMS from …” alerts.
- Vizzy will answer your number conversationally by SMS.
- Spam messages will still be blocked.
- Real customer inbound SMS can still trigger alerts and auto-replies.

Technical notes
- Minimal safe change: no database migration needed.
- Best guardrail is number normalization in both functions so `+14165870788`, `14165870788`, and `4165870788` are treated the same.
- The main bug is logic, not infrastructure:
  - self-alerting in sync
  - explicit CEO skip in auto-reply
- I would preserve current spam filtering and most dedupe/rate-limit logic, only carving out a safe CEO conversation path.
